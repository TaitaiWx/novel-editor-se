# 产品收敛方案

## 目标

把产品主路径收敛为：

1. 打开作品
2. 新建章节
3. 开始写作
4. 再使用大纲 / 人物 / 设定 / AI / 版本

核心原则：

- 主路径优先于功能丰富度
- 小说作者心智优先于文件系统心智
- 渐进暴露优先于一次性堆满能力
- 章节是一等对象，不只是被识别的文本模式

## 当前问题

1. 首次进入时，系统把“恢复目录 / sample-data / 设置 / AI / 同步”等入口混在一起，用户不知道先做什么。
2. 左侧仍然是偏 VS Code 的“资源管理器”，而不是作者心智里的“作品目录 / 章节列表”。
3. 当前“新建文件”不等于“新建章节”，章节创建、排序、重命名、归档没有成为一条明显主线。
4. 右侧面板太早暴露大纲、情节板、三签法、人物、设定等高级能力，新手进入时负担偏大。

## 收敛后的信息架构

### 首页

无标签页时，显示开始页：

- 无工作区
  - 打开作品目录
  - 空白草稿
  - 查看示例项目
- 已打开工作区
  - 新建章节
  - 导入文稿
  - 切换作品目录

### 左侧

左侧标题改为“作品目录”，不强调“资源管理器”。

一级操作：

- 新建章节
- 新建文件
- 新建目录
- 导入文稿
- 搜索文件

其中“新建章节”必须始终可见，不能只藏在右键菜单和图标里。

### 中间

默认中心仍是正文编辑区，但无标签页时先展示开始页。

### 右侧

右侧保留，但在首页态隐藏。
正文未建立前，不让右侧成为主要注意力来源。

## 章节主路径

### Phase 1

先把章节做成一条清晰可走通的主路径：

1. 点击“新建章节”
2. 自动生成文件名，例如 `第01章 未命名.md`
3. 自动写入章节模板
4. 自动打开该章节开始编辑

章节模板建议：

```md
# 第01章 未命名

```

### Phase 2

把章节维护能力做完整：

1. 重命名章节
2. 拖拽排序章节
3. 复制章节
4. 归档章节
5. 批量创建章节骨架

### Phase 3

让“章节”成为比“文件”更高层的对象：

1. 左侧默认按章节视图展示
2. 可选切到原始文件视图
3. 章节列表支持状态、字数、更新时间
4. 章节可以映射到大纲和情节板

## 高级功能的降噪策略

以下能力不删除，但降级为次要入口：

- AI 助手
- 人物
- 设定
- 情节板 / 三签法
- 账号
- 云同步

原则：

1. 先有正文，再引导用户使用这些能力
2. 没有正文时，不把这些能力作为首页主入口

## 本次已经落地的第一阶段

1. 首次启动不再默认打开 sample-data
2. 新增首页式开始页
3. 左侧标题从“资源管理器”收敛为“作品目录”
4. 新增一级操作“新建章节”
5. 新建章节会自动生成默认文件名和章节模板，并自动打开
6. 首页态隐藏右侧高级面板与状态栏，先聚焦开始写作

## 后续建议优先级

P0：

1. 章节拖拽排序
2. 章节重命名体验优化
3. 新建作品向导（创建目录 + 初始结构）

P1：

1. 左侧加入“章节视图 / 文件视图”切换
2. 批量创建章节骨架
3. 章节状态（草稿 / 修订 / 定稿）

P2：

1. 大纲、情节板与章节的双向联动
2. 角色 / 设定与章节引用关系
3. AI 按章节上下文工作，而不是直接面向全量文件树
